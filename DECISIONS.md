| 19 | 2026-04-25 | agent       | Retrieved context goes in the per-turn user message, NOT the system prompt. | CLAUDE.md §6 lists retrieved context as a system-prompt block, but every-turn changes there would invalidate Anthropic's prompt cache (system prompt is the cached prefix). | 20 | 2026-04-25 | privacy     | Embeddings run on-device. Bundle Snowflake `arctic-embed-xs` INT8 ONNX (~23 MB, 384-dim, MTEB retrieval 50.15) under `assets/models/arctic-embed-xs/`; load via `flutter_onnxruntime` 1.7.0; tokenize via `dart_wordpiece` 1.2.0 (pure Dart). | True offline-first — no network for indexing or retrieval, only the LLM call leaves the device. arctic-embed-xs beats vanilla MiniLM-L6 on retrieval at the same size and is a drop-in (same WordPiece vocab). CLS pooling (per the model's `1_Pooling/config.json`), L2-normalized so cosine ≡ dot product matches sqlite-vec's `vec_distance_l2`. Asymmetric model: queries get the prefix `"Represent this sentence for searching relevant passages: "`, documents don't — `EmbeddingProvider.embed(kind:)` parameterizes this. Considered Voyage AI's hosted embedding API for faster shipping but rejected: the moat is offline access to a private corpus. Considered the `sqlite_vector` wrapper or `fonnx` for an all-in-one stack; rejected because `fonnx` is unmaintained on pub.dev and `sqlite_vector` ties to its own DSL. | —          |
