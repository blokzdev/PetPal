| 26 | 2026-04-25 | build       | Local self-verification must run the **exact** CI command (`flutter analyze --fatal-infos`) and check the exit code, not eyeball-grep truncated output. Root cause: an unused `import 'package:flutter/material.dart';` in `test/app/screens/wiki_browser_test.dart` introduced in task 2.7. Local checks ran `flutter analyze \| tail -3`, which truncated the warning above the cut and showed only the trailing "1 issue found" line — easy to miss in a multi-line summary. CI uses `flutter analyze --fatal-infos` and exits 1, which is what surfaced the failure. | 27 | 2026-04-26 | ui          | Two permanent design rules adopted in Phase 3.5 alongside `PRODUCT.md` + `VOICE.md`. **(a) Internal/external vocabulary split.** `SOUL.md`, "wiki", "skill", "harness", "agent", "frontmatter", "synthesis/digest", "tool call", and the raw tool names (`read_wiki`, `search_wiki`, `write_wiki_entry`, `update_soul`, `schedule_reminder`) are internal architecture vocabulary — they belong in `lib/harness/`, `CLAUDE.md`, `DECISIONS.md`, code comments, and developer-only screens, but never in user-facing UI. The user-facing translations are codified in `VOICE.md` §3: SOUL → **Profile**, wiki → **Journal**, skill → **Care guide**, frontmatter → **Profile fields**, synthesis/digest → **Weekly summary**, agent/LLM → **PetPal**, and tool pills render the friendly forms ("saving a memory", "checking Loki's journal"). The harness keeps its precise vocabulary so the architecture is readable; the product surface speaks the user's language so the moat (compounding memory of *Loki*) is what's salient. A grep CI lint enforces this in Phase 5. **(b) Pet-name interpolation rule.** Personalization compounds — "Loki's journal" reinforces the moat harder than "Journal" — but over-interpolating reads as scripted, and on a multi-pet dashboard there is no single name to interpolate. The rule: **interpolate** on per-pet page titles + per-pet body copy (Profile, Journal, individual Memory entries, weekly-summary entry titles, photo timeline, per-pet reminders, chat empty state, chat tool pills); **don't interpolate** on global/cross-pet screens (Settings, Care guides browser, API key entry, billing, export confirmations, error banners) or on transient action labels/buttons ("Save", "Cancel", "Edit profile", "Open journal"). Multi-pet dashboards (Phase 5) use a static "My pets" title with names on the cards. The maxim: **page destinations personalize; buttons and global shells stay static.** Both rules are documented in `VOICE.md` §3–§5. | —          |
